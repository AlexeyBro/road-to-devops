# Road to DevOps

## 1. Linux

### Теория
- [ ] Установить Linux на компьютер.
- [ ] Пройти базовый курс по работе с Linux.

---

## 2. Процессы *(топ-1 тема на собеседованиях)*

> Необходимо хорошо изучить теорию.

### Что изучить
- [ ] Команда `ps aux`.
- [ ] Что такое **troubleshooting**.
- [ ] `systemd`:
  - что это такое;
  - уметь писать `Unit File`.
- [ ] OOM Killer.

---

## 3. Мониторинг ресурсов

### Что изучить
- [ ] Утилиты мониторинга:
  - `top`
  - `htop`
  - современные альтернативы (`btop`, `glances`).
- [ ] Научиться читать показатели:
  - `PID`
  - `RES`
  - `VIRT`
  - `SHR`
  - `%CPU`
  - `%MEM`
  - `SI`
  - `WA`
  - и другие.
- [ ] Понимать, что такое **Load Average (LA)**.

---

## 4. Память

### Что изучить
- [ ] Находить процессы, потребляющие много памяти.
- [ ] Понимать причины утечек памяти.
- [ ] Уметь устранять проблемы.

---

## 5. Диск

### Что изучить
- [ ] `df`
- [ ] `du`
- [ ] INODE.
- [ ] Типы файловых систем.
- [ ] Как работает команда `mount`.
- [ ] Базовое понимание RAID-массивов.
- [ ] LVM.

### Практика

Сгенерировать с помощью нейросети несколько различных проектов:

- Backend + Frontend + База данных.
- На разных языках:
  - Python
  - Java
  - Node.js
  - Go
  - и др.

После этого:

- [ ] Развернуть их на локальном Linux.
- [ ] Научиться запускать, останавливать и диагностировать ошибки.
