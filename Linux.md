# Linux

## Базовые команды

Вызывает справочную информацию о команде, где n любая команда

```bash
n --help
```

Вызывает справочную информацию о команде в более структурированном виде, 
чем через --help, где n любая команда

```bash
man n
```

Выполнить предыдущую команду (обычно используется в связке с новой командой)

```bash
!!
```

Команда "Конвейр". ""Берет вывод (результат) команды и передает его как ввод 
для следующей команды.
Позволяет соединять простые команды в цепочки

```bash
|
```

Вывести данные в `stdout` (аналог `print`)

```bash
echo
```

Вывести историю всех команд, где n кол-во выведенных команд с конца

```bash
history n
```

## Работа с папками и файлами

Создать файл (также используется для работы с метаданными файла)

```bash
touch {file_name}
```

Создать папку

```bash
mkdir {folder_name}
```

Создать папку внутри папки флаг -p (parant)

```bash
mkdir -p {parant_folder_name}/{folder_name}
```

Скопировать файл

```bash
cp {file_name} {folder_name}/{copy_file_name}
```

Скопировать несколько файлов с помощью *

```bash
cp -r {folder_name}/{copy_file_name*} {destination_folder_name}
```

Скопировать файл с добавлением только более свежих данных

```bash
cp -u {file_name} {folder_name}/{copy_file_name}
```

Скопировать директорию в другую директорию

```bash
cp -r {folder_name} {cope_folder_name}
```

Переместить или переименовать файл

```bash
mv {file_name} {folder_name}
```

Удалить файл

```bash
rm {file_name}
```

Удалить все файлы с определенным расширением

```bash
rm *.txt
```

Удалить все файлы в директории

```bash
rm *
```

Удалть пустую папку

```bash
rmdir {folder_name}
```

Удалить папку вместе с содержимым, вывести ошибки если будут -r (recursive)

```bash
rm -r {folder_name}
```

Удалить папку вместе с содержимым принудительно -f (force)

```bash
rm -rf {folder_name}
```

## Ссылки

Выделяют два типа ссылок:
- Символические ссылки - по сути ярлыки на файлы и папки, при удалении файла/папки
на которую ссылается ярлык сам ярлык остается, но становится нерабочим
- Жесткие ссылки - это обычные ссылки на область памяти, где находится файл. При
создании каждого файла для него сразу создается 1 жесткая ссылка

Создать символическую ссылку на папку/файл

```bash
ln -s {path_folder/file} {link_name}
```

Создать жесткую ссылку, особенность в том что при изменении оригинала 
или дубликата оба файла будут меняться, так как ссылки указывают на одну область
в памяти, удобно например для сохранения копий важной инфы

```bash
ln {path_file} {link_name}
```

## Работа через `sudo`

`sudo` (*Super User Do*) позволяет выполнять команды с правами администратора.

Перейти в режим администратора

```bash
sudo su
```

или

```bash
sudo su {user_name}
```

Переключиться на другого пользователя

```bash
su {user_name}
```

Изменить владельца файла

```bash
sudo chown {user}:{group} {file_name}
```

> `chown` — *Change Owner*

Изменить права доступа

```bash
sudo chmod {permissions} {file_name}
```

> `chmod` — *Change Mode*

## Работа с пользователями и группами

### Пользователи

Список источников информации о пользователях хранится в **/etc/nsswitch.conf**  
Данные о пользователях хранятся в **/etc/passwd**  
Пример: *root:x:0:0:root:/root:/bin/bash*, где:  
- root - имя пользователя
- x - указатель на файл с хешем пароля etc/shadow (раньше пароль хранился в файле
/etc/passwd, но тк доступ к этому файлу есть у всех пользователей, то можно было
взять хэши паролей, по этому их вынесли в файл, который доступен только для /root
- 0 - User ID (UID)
- 0 - Group ID (GID)
- root - GECOS информация о пользователе, например телефон или фамилия
- /root - домашний каталог пользователя
- /bin/bash - командная оболочка, которая запускается для пользователя

Могут быть разные варианты командных оболочек: 

```bash
/bin/sh
/bin/bash
/bin/dash
/bin/rbash
/bin/zsh
/usr/bin/fish
```

Также вместо программной оболочки может запуститься программа,
которая сразу завершит возможность работать с shell

```bash
/usr/sbin/nologin
```

Для того чтобы у нового пользователя сразу были доступны какие-то файлы
их можно положить в */etc/skel/*

Добавить нового пользователя (примитивно, без домашней деректории и тд)

```bash
useradd {user_name}
```

Добавить нового пользователя с ручной настройкой

```bash
useradd {user_name} -m -d {home_dir} -k {dir_for_initial_data} -s {bash_name}

-m - создать домашний каталог
-d - где будет находится домашний каталог (если не указано, то /home/{user_name})
-k - перенести данные из этого каталога в домашний каталог пользователя
-s - какой shell будет использоваться
```

Задать пароль для нового пользователя

```bash
passwd {user_name}
```

Добавить нового пользователя с нужными полями заполненными по умолчанию

```bash
adduser {user_name}
```

Отредактировать поля пользователя

```bash
usermod {user_name} {keys}
```

Удалить только пользователя

```bash
userdel {user_name}
```

Удалить пользователя со всеми его данными

```bash
userdel -r {user_name}
```

Заблокировать пользователя

```bash
usermod -L {user_name}
```

Разблокировать пользователя

```bash
usermod -U {user_name}
```

### Группы

Список всех групп находится в **/etc/group**  

Посмотреть какие в какие группы входит пользователь

```bash
groups {user_name}
```

Создать группу

```bash
addgroup {group_name}
```

Изменить главную группу

```bash
usermod {user_name} -g {group_name}
```

Изменить вторичную группу

```bash
usermod {user_name} -G {group_name},{group_name}...
```

Добавить вторичную группу к уже присвоенным

```bash
usermod {user_name} -a -G {group_name},{group_name}...
```

Удалить группу (нельзя удалить группа, если она является 
главной для хотя бы одного пользователя)

```bash
groupdel {user_name}
```

## Избирательные права доступа (DAC)

| Код | Право | Обозначение | Для файлов | Для директорий |
|-----:|--------|-------------|-------------|-------------|
| 4 | Read | `r--` | Чтение | Чтение |
| 2 | Write | `-w-` | Запись | Запись |
| 1 | Execute | `--x` | Исполнение | Просмотр вложенных директорий |

Права складываются:

| Код | Права |
|-----:|--------|
| 4+2 = 6 | `rw-` |
| 4+1 = 5 | `r-x` |
| 4+2+1 = 7 | `rwx` |

Права задаются для трех категорий пользователей:

1. Владелец (Owner)
2. Группа (Group)
3. Остальные пользователи (Others)

Пример:

```bash
chmod 764 {file_name}
```

Расшифровка:

```
7 = rwx (читать/записывать/исполнять)
6 = rw- (читать/записывать)
4 = r-- (читать)
```

Задать права на все файлы внутри директории

```bash
chmod -r 764 {dir_name}
```

Защитить файл/директорию от удаления с использованием sticky bit

```bash
chmod 1XXX {file_name}/{dir_name}
```

Изменить владельца и группу файла/директории

```bash
chown {new_owner_name}:{new_group_name} {file_name}
```

Изменить только владельца файла/директории

```bash
chown {new_owner_name} {file_name}
```

Изменить только группу файла/директории

```bash
chown :{new_group_name} {file_name}
```

## Пакетный менеджер

### dpkg

Низкоуровневый установщик пакетов **dpkg**. Устанавливает только требуемый пакет,
который должен лежать локально, не подтягивает пакеты из удаленного репозитория.
Как правило требуется только для того чтобы исправить появившиеся проблемы с
пакетами, но в штатном режиме не используется

Установка пакета

```bash
dpkg -i
```

Удаление пакета без конфигурационных файлов

```bash
dpkg --remove
```

Полное удаление пакета

```bash
dpkg --purge
```

Вывести список пакетов в системе

```bash
dpkg --list
```

Узнать информацию о пакете

```bash
dpkg --list {package_name}
```

Узнать подробную *информацию* о пакете

```bash
dpkg --status {package_name}
```

Посмотреть полный перечень файлов, которые добавлены в рамках установки пакет

```bash
dpkg --statusfiles {package_name}
```

### Основной пакетный менеджер в Линукс - **apt**

Установщик **apt** - обертка над **dpkg**, которая анализирует зависимости устанавливаемых
пакетов и обновляет/скачивает необходимые данные. Может скачивать и устанвливать
пакеты из удаленных репозиториев, ссылки на которые находятся в /apt/sources.list

```bash
apt - advanced package tool, является надстройкой над низкоуровневым dpkg
```

Найти пакет

```bash
apt search {package_name}
```

Подробное инфо о пакете

```bash
apt show {package_name}
```

Скачать пакет без установки

```bash
apt download {package_name}
```

Установить программу

```bash
sudo apt install {package_name}
```

Удалить программу

```bash
sudo apt remove {package_name}
```

Обновить версию пакета

```bash
sudo apt upgrade {package_name}
```

Обновить все установленные пакеты безопастно

```bash
sudo apt upgrade
```

Обновить все установленные пакеты с принудительным удалением старых версий,
если это потребуется

```bash
sudo apt full-upgrade
```

## Поиск

Поиск файла по имени (с учетом регистра)

```bash
find . -type f -name "{file_name}"
```

Поиск файла без учета регистра

```bash
find . -type f -iname "{file_name}"
```

Поиск папки

```bash
find . -type d -name "{directory_name}"
```

Поиск с использованием шаблона

```bash
find . -type f -name "file.*"
```

Поиск по правам доступа

```bash
find . -type f -perm 0664
```

Поиск по размеру

Больше 10 МБ

```bash
find . -size +10M
```

Меньше 10 МБ

```bash
find . -size -10M
```

Найти все файлы, кроме указанного

```bash
find . -type f -not -name "{file_name}"
```

# Bash Shell

Все команды и скрипты выполняются через **Shell**.

**Shell** — это программа-посредник между пользователем и операционной системой.

Популярные реализации Shell:

- Bash
- Zsh

### Что делает Shell

- принимает команды пользователя;
- разбирает их;
- запускает нужные программы;
- получает результат выполнения;
- выводит результат обратно в терминал.

Схема работы:

```
Пользователь
      │
      ▼
Терминал
      │
      ▼
Shell (bash/zsh)
      │
      ▼
Ядро Linux
      │
      ▼
Программы
```

## Bash-скрипты

Любой Bash-скрипт начинается с шебанга:

```bash
#!/bin/bash
```

Пример:

```bash
#!/bin/bash

echo "Hello"

mkdir test

cd test

touch file.txt
```

Запуск скрипта

```bash
bash {script_name}.sh
```

или после выдачи прав на выполнение

```bash
chmod +x {script_name}.sh
./{script_name}.sh
```

## Перенаправление ввода/вывода

Оператор, который перезаписывает данные при добавлении, если файла нет, то
создаст файл

```bash
{command} {file_name} > {result_file_name}
```
Пример
sort names.txt > sorted_names.txt

Оператор, который добавляет данные, без перезаписи

```bash
{command} {file_name} >> {result_file_name}
```

Оператор для перенаправления ошибок

```bash
2> или 2>>
```

С помощью оператора перенаправления можно также создать файл

```bash
cat>file.txt
```

С помощью обратной стрелочки можно выводить данные с командой grep

```bash
grep {search_text}<file.txt
```

Команда сделай, если предыдущее действие успешно &&

```bash
grep {search_text}<file.txt && echo {some_text}
```

Команда сделай, если предыдущее действие НЕ успешно ||

```bash
grep {search_text}<file.txt || echo {some_text}
```

## Монтирование файловых систем

Синтаксис команды для монтирования, где: 
-k опции, 
-t тип устройства (не всегда используется)

Опции:
- r (read only) в монтируемый раздел доступен только для чтения

```bash
mount -t {system_type} -k {device} {point_of_mount}
```

Размонтировать систему

```bash
гmount {point_of_mount}
```
